# Arabic Grammar Pack

Phase 10 adds a separate read-only SQLite pack for Arabic i'rab in the full Word Study route.

## Source and attribution

- Content: Quranic Arabic Corpus Arabic i'rab prose, packaged by the `zeeyado/quran-ebook` grammar-dictionary project.
- Artifact: `quran_irab_v1.4.zip`.
- Source URL: `https://github.com/zeeyado/quran-ebook/releases`.
- Source artifact SHA-256: `804b7397d1a239cdeace01b3ea9a656342062f582350c52930b0481dbd3300fb`.
- Imported coverage: 5,785 ayahs and 29,881 Arabic analysis passages. Uncovered ayahs render an explicit unavailable state.
- Permission and qualified Quranic Arabic review remain release requirements; this implementation records provenance but does not represent either sign-off.

No attribution block is rendered in the Grammar tab. Attribution and provenance are maintained in this document and the generated pack manifest.

## Rebuild

Place the source archive at `.artifacts/word-grammar/quran_irab_v1.4.zip`, or set `WORD_GRAMMAR_SOURCE`, then run:

```bash
npm run compile:word-grammar-pack
npm run test:word-study-pack
```

Generated files:

- `dist/word-grammar-packs/qac-irab-v1.4/quran-word-grammar.db`
- `dist/word-grammar-packs/qac-irab-v1.4/manifest.json`

The compiler parses StarDict entries, removes presentation markup, retains the original Arabic headings and prose, maps chapter names to canonical `surah:ayah` keys, and emits deterministic ordered passages.

## Runtime behavior

- The pack opens lazily through Expo SQLite when the Grammar tab is selected.
- Selected-word matching is presentation-only: the app normalizes the complete selected surface and shows a match only when an entire word in the Arabic heading has the same normalized form. Morpheme fragments do not qualify.
- The original verse-level passage order remains unchanged.
- The complete ayah analysis stays collapsed until requested and always contains every source passage in its original order.
- No grammar prose is generated by the app or an LLM.
